## 2024-06-25 - [DataModal Filtering and Sorting Micro-optimizations vs Schwartzian Transform]
**Learning:** While replacing array methods (`.filter()`, `.map()`, `.some()`) with manual `for` loops and pre-allocated arrays provides raw execution speedups in micro-benchmarks for large datasets, it severely degrades code readability and is flagged as a prohibited micro-optimization in this codebase. However, algorithmic improvements like the Schwartzian Transform (decorate-sort-undecorate) for sorting are highly effective and acceptable because they fundamentally change the O(N log N) overhead of repeated key calculations inside sort comparators without destroying declarative code structures elsewhere.
**Action:** Always prioritize algorithmic improvements (like Schwartzian transforms for sorting or hash maps for lookups) over raw manual loop unrolling. Leave native JS array iteration methods intact unless the algorithm itself necessitates a change in data structure.
