## 2025-05-15 - [Profile Admin Stats Optimization]
**Learning:** [Dashboard chart data logic originally used an O(N*M) nested looping approach with expensive Date().toISOString() parsing for every single item over potentially 1000 reconciliations fetched. Replacing declarative array methods (.filter().map()) and nested loops on large dynamic data with a single-pass hash map aggregation combined with string slicing for ISO dates significantly cut operations down to O(N + M) providing massive performance gains.]
**Action:** [Use hash map lookups and single-pass accumulation loops over nested iteration for array-of-objects grouping logic whenever rendering complex analytics views or stacked chart data. Furthermore, always avoid full Date parsing inside loops for static string extraction.]
