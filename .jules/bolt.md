## 2024-05-10 - O(N log N) String Manipulation in Array Sorting is a Bottleneck
**Learning:** In components rendering and sorting large datasets (e.g., `DataModal.jsx` handling ~10k items), calling expensive operations like string replacements (`.replace(/,/g, '')`) and floating-point conversions (`parseFloat`) directly inside `Array.prototype.sort()`'s comparator function is a massive performance bottleneck. Because the comparator is called O(N log N) times, these heavy string conversions are unnecessarily repeated for the same elements.
**Action:** When implementing custom sorting logic that requires string mutation or parsing before comparison, apply a Schwartzian Transform (map-sort-map). Pre-calculate the parsed values into a temporary mapped array (O(N)), perform the sort using those cached values (O(N log N) but with cheap, direct comparisons), and map back to the original objects. This improves sort times for large datasets by approximately ~6-7x.
