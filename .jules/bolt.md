## 2024-05-18 - Schwartzian Transform for Fast String Parsing in Sort
**Learning:** The `DataModal.jsx` component suffered from an $O(N \log N)$ performance bottleneck during column sorting. This was caused by executing expensive operations—specifically string replacement (`replace(/,/g, '')`) and numeric conversion (`parseFloat`)—directly inside the `Array.prototype.sort()` callback. Because the sort callback is executed repeatedly during value comparison, the parsing logic ran redundantly.
**Action:** Implementing a Schwartzian transform (Decorate-Sort-Undecorate) extracts these heavy parsing operations from the sort loop. By mapping the array to pre-calculate and cache the parsed values once per row ($O(N)$), the $O(N \log N)$ sorting phase only performs fast, pure variable comparisons. Use this pattern whenever sorting requires complex derivation or parsing of values in JavaScript.
