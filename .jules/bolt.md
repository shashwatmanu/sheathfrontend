## 2024-05-15 - [Vortex Canvas Render Loop Optimization]
**Learning:** In high-frequency canvas render loops (like the `Vortex` component rendering 700 particles per frame), repeatedly calling `ctx.save()`, `ctx.restore()`, and setting static properties (like `ctx.lineCap`) inside the inner per-particle loop incurs significant rendering overhead. Extracting these to wrap the entire loop and setting static properties once yields substantial performance gains, dramatically reducing CPU time per frame.
**Action:** When auditing canvas animation loops, identify static context states (`lineCap`, `globalAlpha` if constant, etc.) and state backup/restore (`save()`/`restore()`) operations inside `for` loops. Move them outside the loop and wrap the entire draw operation, but ensure you do not combine `save()`/`restore()` across multiple distinct compositing steps to prevent canvas context leakage.
